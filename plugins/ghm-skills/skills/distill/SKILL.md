# Distill — 会话经验提炼器

回顾当前会话中的试错过程，提取可复用知识，确保同样的弯路不再重走。

## 核心流程

### 第一阶段：扫描试错模式

遍历完整对话历史，识别**摩擦点**——即成功路径不是一条直线的地方。关注以下信号：

- **报错→修复循环**：命令/代码执行失败后被修正
- **方案切换**：方案 A 尝试后被放弃，改用方案 B
- **反复尝试**：同一操作以不同变体多次执行
- **非预期行为**：某些东西表现与预期不同，需要排查
- **知识空白**：信息在任务中途才被查到或发现
- **配置发现**：不显而易见的设置、参数或环境变量

对每个摩擦点，提取：
1. **最初的尝试**（错误的/次优的做法）
2. **失败原因**（报错信息、非预期结果、或放弃的原因）
3. **最终方案**（实际有效的做法）
4. **根因**（为什么第一次尝试会失败——缺少上下文？错误假设？版本问题？）

### 第二阶段：识别正向模式（次要）

同时记录那些一次成功但值得留档的非显然技巧：
- 涉及项目特有约定，难以凭直觉猜到
- 使用了不常见的 API 用法、参数或配置
- 代表一种可复用的工作流模式

此部分从简。重心在摩擦点。

### 第三阶段：决定写入目标

对每条经验，判断最佳写入位置：

| 经验类型 | 写入目标 | 示例 |
|---|---|---|
| 跨项目可复用 | `~/.claude/skills/` 下**新建 skill** | "这个包必须加 `--legacy-peer-deps`" |
| 仅当前项目可复用 | `.claude/skills/` 下**新建 skill** | "此仓库中需先跑 migration 再 seed" |
| 已有 skill 需补充 | **更新 skill** | 给已有 skill 补充边界情况或注意事项 |
| 项目特定事实/约定 | **CLAUDE.md** | "开发服务器跑在 3001 端口，不是 3000" |
| 文档缺失 | **README 或其他文档** | 缺少的安装步骤导致了困惑 |

**优先级**：Skill（新建或更新）> CLAUDE.md > 其他文档。

判断标准："这条经验在*另一次会话*、*另一个任务*中是否有帮助？"
如果是 → skill；如果只是一次性的项目事实 → CLAUDE.md。

### 第四阶段：展示发现

输出结构化报告：

```
## Distill 报告

### 发现摩擦点：N 个

#### 1. [简短标题]
- **尝试了**：[最初的做法]
- **失败原因**：[根因]
- **解决方案**：[最终有效的做法]
- **行动**：新建 skill `skill-name` / 更新 CLAUDE.md / 更新 `path/to/doc`

#### 2. ...

### 正向模式：N 个
#### 1. [简短标题]
- **做法**：[具体操作]
- **价值**：[为什么值得记录]
- **行动**：...

### 拟议变更

[对每项行动，展示将创建的完整文件内容或修改的 unified diff]
```

展示新文件的完整内容，以及修改项的 unified diff。

### 第五阶段：确认后执行

**在用户明确确认之前，不要写入任何文件。**

用户审阅并批准后（可能带修改意见），执行变更。
若用户只批准部分，则仅执行被批准的部分。

## 提取 Skill 的写作规范

从经验中创建新 skill 时：

- **命名**：小写加连字符，描述性强，最长 64 字符
- **描述**：包含具体触发条件——报错信息、工具名、症状表现。
  写得"主动"一些，确保相关场景下 skill 确实被激活。
- **正文结构**：
  ```
  ## 场景
  [什么情况下需要这个知识]

  ## 正确做法
  [分步骤的具体操作，可直接照做]

  ## 原理
  [为什么这样做是对的——让读者理解背后的逻辑]
  ```
- 语气聚焦于"应该怎么做"，直接给出正确路径，而非罗列禁忌。

## CLAUDE.md 更新规范

- 追加到最相关的已有小节，或新建 `## 经验教训` 小节
- 使用祈使句："先执行 X 再执行 Y"，而不是"我们发现应该先执行 X 再执行 Y"
- 简要说明原因："先执行 X 再执行 Y——否则 Z 会静默失败"

## 边界情况

- **没有发现摩擦点**：如果会话一帆风顺没有重试，如实说明。可选择性记录正向模式，但不要强行提取。
- **摩擦源于偶发因素而非知识缺口**：如果重试是因为网络波动、服务暂时不可用等不可复现的偶发问题，跳过——没有可编码的知识。
- **经验已存在**：在提议前检查已有 skill 和 CLAUDE.md，避免重复。如果相关 skill 已存在但不完整，提议更新而非新建。
