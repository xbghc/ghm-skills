# Stitch 提示词编写指南

本指南基于 [Stitch 官方提示词指南](https://discuss.ai.google.dev/t/stitch-prompt-guide/83844) 及社区实践经验整理。在为用户编写 Stitch 生成提示词时，遵循以下原则。

## 核心原则

1. **每次只做一到两处修改**——Stitch 在接收多个变更时容易丢失已有布局
2. **使用自然语言**——不需要 XML 或 JSON 结构，简短清晰的自然语言效果最好
3. **明确指向具体元素**——使用 UI/UX 术语（如"导航栏"、"主按钮"、"卡片布局"、"Hero 区域"）
4. **先粗后细**——复杂应用先用概括性描述生成整体，再逐屏逐组件细化

## 起步：概括 vs 详细

根据设计阶段选择提示词粒度：

**概括型**（用于头脑风暴或复杂应用初始阶段）：
> "一个面向马拉松跑者的应用。"

**详细型**（用于获得更精确的起始结果）：
> "一个面向马拉松跑者的应用，包含社区互动、寻找跑步搭档、训练建议和附近赛事查找功能。"

## 用形容词定义视觉氛围

形容词直接影响 Stitch 选择的配色、字体和图像风格：

> "一个充满活力、鼓舞人心的健身追踪应用。"

> "一个极简、专注的冥想应用。"

## 针对特定页面/组件的提示词

聚焦具体屏幕，描述清楚内容和风格：

> "日式工装风格男士运动服饰的产品详情页。深色极简设计，深蓝色主色调。服装图片少量展示，天然面料，不浮夸。"

> "茶具店的产品详情页，日式侘寂风格。售卖草本茶和陶瓷器。中性极简配色，黑色按钮。柔和优雅的字体。"

## 修改已有设计

### 布局和组件修改

明确说明**改什么**和**怎么改**：

> "在首页顶部的 header 中添加一个搜索栏。"

> "将登录页的主操作按钮改大，使用品牌主蓝色。"

### 颜色修改

可以指定具体颜色或描述色调氛围：

> "将主色调改为森林绿。"

> "更新为温暖、有亲和力的配色方案。"

### 字体和边框

> "使用活泼的无衬线字体。"

> "将所有按钮改为全圆角。"

> "输入框使用 2px 实线黑色边框。"

### 图片修改

明确指定目标图片的位置和修改内容：

> "将落地页上所有产品图片的背景改为浅灰褐色。"

> "在'团队'页面中，'Dr. Carter（首席牙医）'的照片：将她的白大褂改为黑色。"

修改主题色时，如果希望图片也跟随变化，需要显式说明：

> "将主题更新为浅橙色。确保所有图片和图标也匹配这个新配色方案。"

### 语言切换

> "将所有产品文案和按钮文字切换为中文。"

## 社区实践经验

### 长提示词的问题

超过 5000 字符的提示词容易导致 Stitch 遗漏部分组件。解决方案：

- 先用简短提示词生成基础结构
- 再逐步添加细节，每次一到两处修改

### 编辑时的重置问题

Stitch 在编辑时可能重建整个布局，导致之前的修改丢失。应对策略：

- **严格遵循单次单改原则**——不要在一个提示词中混合布局修改和 UI 组件添加
- **每次成功后保存截图**——如果后续修改导致重置，可以作为参考重新描述

### 推荐的迭代节奏

1. 用简短提示词生成整体结构
2. 确认结构正确后，逐个添加组件
3. 组件到位后，调整样式细节（颜色、字体、间距）
4. 最后处理图片和视觉细节
